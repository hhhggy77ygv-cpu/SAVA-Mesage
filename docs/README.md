# Документация SAVA Messenger

Этот файл описывает основные шаги для настройки, разработки и загрузки проекта на GitHub.

## Установка

В корне проекта выполните:

```bash
npm install
```

Если вы используете Windows и сталкиваетесь с проблемами, убедитесь, что установлены последние версии Node.js и npm.

## Локальная разработка

Запустите сервер и веб-клиент вместе:

```bash
npm run dev
```

Для запуска только десктопного приложения используйте:

```bash
npm run dev:desktop
```

## Сборка

Сборка для серверной и веб-части:

```bash
npm run build
```

Сборка Windows-десктопа:

```bash
npm run build:desktop
```

## Файлы и структура

- `apps/server` — серверная часть
- `apps/web` — веб-клиент
- `apps/desktop` — Electron-приложение
- `certs` — SSL-сертификаты
- `sounds` — медиа-файлы

## Подготовка к GitHub

Проект уже содержит:

- `.gitignore` — исключает `node_modules`, файл `.env`, сборочные каталоги и личные данные
- `.gitattributes` — нормализует переводы строк и конфигурацию текстовых файлов

Перед публикацией проверьте, что в коммите нет личных данных:

- секретных ключей
- `.env`
- `apps/server/uploads`
- локальных сертификатов

## Команды Git

1. Инициализация репозитория (если ещё не сделано):

```bash
git init
```

2. Добавление файлов и первый коммит:

```bash
git add .
git commit -m "Initial project setup"
```

3. Добавление удалённого репозитория GitHub:

```bash
git remote add origin https://github.com/username/repo.git
```

4. Отправка на GitHub:

```bash
git push -u origin master
```

## Советы

- Не добавляйте в репозиторий личные сертификаты и `.env`
- Используйте `git status` перед каждым коммитом
- Держите документацию в `README.md` и `docs/README.md` в актуальном состоянии
